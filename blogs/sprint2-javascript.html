JavaScript in Real Terms

Last week we began learning JavaScript - “the language of the internet”. JavaScript controls much of the interactivity we are used to on the internet. In this blog, we’ll take a look at what JavaScript is in plain English.

HTML and CSS

Before we can look at what JavaScript does, we need to have a quick refresher about HTML and CSS. JavaScript often interacts with HTML and CSS files to create dynamic content and visual styles that change and interact with the user.

HTML contains the content of the webpage - the text, images, links and so on - while the CSS contains instructions about how the content should be displayed. It describes the fonts, colours, positions and other visual qualities about the content elements.

During this sprint’s reflections, I came up with a story to describe their relationship. I told the tale of a young boy who, eager to earn a coin from his mother for completing a chore, ran off to the tailor’s shop with only half of the instructions for the clothes she wanted made.

The tailor looked at the boy and said I can’t do anything with this - I know she needs a dress and a blouse and so on, but what fabric? What specifications? What colour? Control Flow and Loops

Most web pages contain scripts written in JavaScript that execute to handle all manner of interactions. When a script runs, it executes its commands in sequential order. This order is called the control flow.

We can also use loops to execute parts of the control flow again and again until a certain condition is reached. Avoiding technical jargon, let’s look at the “control flow” for walking to the shops.

  1. If you’re in your house, leave your house.
  2. Point yourself towards the supermarket.
  3. Take a step forward.
  4. If you are not at the supermarket, go back to step 3.
  5. Enter the shop.


In this flow, a loop executes at steps 3 and 4. As long as the condition of being at the supermarket is not yet reached, you continue to loop that part of the code, which leads to you walking.

Relative

The element will take its normal place in the flow of the document, but you can move it around relative to this position.

Using the commands top, right, bottom and left, you can add space on each side of the element that will push it around on the page. Here, the code left: 100px; is used to add 100 pixels of space to the left of where it was previously.

You can also use the command z-index to determine the depth of the element. Where two elements overlap each other, the one with the higher z-index will appear on top.

The DOM

Walking to the shops is fine, but we want our scripts to interact with our website elements. This happens through the Document Object Model (DOM).

This is an interface that maps out all of the elements on a website into a tree. The viewing window contains the document itself, which contains the body of the document, which contains all the headings and images, and so on.

Elements in the DOM can be “grabbed” by your JavaScript. After you get a hold of these elements, you can do all manner of things to them. For example, you can “grab” all of the images in your document and then run a script that turns them upside down when the user clicks a button.

Arrays and Objects

JavaScript lets you create arrays and objects that take a whole bunch of values, give them names and store them under one heading. These are very useful for describing things in the real world.

Think about your kitchen cupboard. You could create a JavaScript object called cupboard() that contains items like plates: 3 and cups: 10. Then, you can simply access cupboard.plates to know how many plates we have (this would return 3).

Arrays are similar, but instead of giving the values names, you simply have a bunch of values with an index. You can imagine this like a database of phone numbers.

In the array phone-numbers[‘021123456’, ‘021234567’, ‘021345678’] we have three numbers that we can access. The only trick is that it starts counting at 0. So, the first number is indexed at 0, the second at 1, and so on.

In this case, asking for phone-numbers[2] gives us the value ‘021345678’.

Functions

Functions are where JavaScript gets really good. A function is just a block of code that gets a name. When you call that name in your script, it executes the function. For example, you could call a function that displays a message saying “hello”. But that’s no fun.

Functions can be as complicated as you like. They drive browser games, search large databases for information and display complicated animations. The only limits are your imagination - and your coding ability.
